# Providers

> AIDEN ne vous enferme pas. Claude pour l'analyse, Gemini pour le code, Copilot en backup gratuit, Ollama pour le 100% local — mixez comme vous voulez.

## Decision rapide

> **Vous debutez ?** → Claude Code.
> **Budget serre ?** → Gemini.
> **Donnees sensibles ?** → OpenCode + Ollama.
> **Deja Copilot ?** → Ajoutez-le en fallback.

## Providers supportes

AIDEN supporte 4 providers CLI nativement et permet de brancher n'importe quel LLM via API :

| Provider | Type | Forces | Context | Cout |
|---|---|---|---|---|
| [Claude Code](/fr/guides/providers/claude-code) | CLI | Analyse, review, long contexte | 200k | $$$ |
| [Gemini CLI](/fr/guides/providers/gemini) | CLI | Generation code, multi-fichier | 1M | $ |
| [OpenCode](/fr/guides/providers/opencode) | CLI | Open source, multi-backend | Variable | Variable |
| [GitHub Copilot](/fr/guides/providers/copilot) | CLI | Quick fix, completion | 50k | Gratuit |
| [API custom](/fr/guides/providers/custom-api) | HTTP | N'importe quel LLM | Variable | Variable |

## Quel provider choisir ?

**Vous debutez ?** Commencez avec [Claude Code](/fr/guides/providers/claude-code) — c'est le provider par defaut, le plus polyvalent.

**Vous voulez optimiser les couts ?** Combinez Claude Code (analyse) + [Gemini](/fr/guides/providers/gemini) (generation) avec le routage `auto`.

**Vous avez des contraintes de securite ?** Utilisez [OpenCode](/fr/guides/providers/opencode) + Ollama pour un pipeline 100% local.

**Vous avez deja GitHub Copilot ?** Ajoutez-le comme [fallback gratuit](/fr/guides/providers/copilot) dans votre config multi-provider.

**Vous avez un proxy LLM interne ?** Configurez un [provider API custom](/fr/guides/providers/custom-api) (LiteLLM, Azure, OpenAI).

## Configuration rapide

Dans `.aiden/aiden.config.yaml` :

```yaml
provider: claude-code
providers:
  claude-code:
    enabled: true
    command: claude
    model: claude-sonnet-4-6
    maxTokens: 100000
    costPer1k: 0.003
  gemini:
    enabled: true
    command: gemini
    model: gemini-2.5-pro
    maxTokens: 1000000
    costPer1k: 0.001
```

> **Information** : Consultez le guide de chaque provider pour les instructions d'installation et d'authentification.

## Routage

Le routage intelligent est ce qui rend AIDEN multi-provider vraiment utile. Au lieu de tout envoyer au meme LLM, chaque tache va au provider le plus adapte — automatiquement.

4 strategies disponibles :

- **prefer** (defaut) : utilise le provider prefere, fallback si indisponible
- **auto** : choisit le meilleur provider selon la tache
- **fixed** : un seul provider toujours
- **cost-optimized** : minimise les couts

```yaml
routing:
  strategy: auto
  fallback: [claude-code, gemini, opencode, copilot]
```

En mode `auto`, chaque agent recoit le provider optimal :

| Tache | Provider par defaut |
|---|---|
| Analyse, Planning, Documentation | claude-code |
| Generation code, Tests | gemini |
| Quick fix, PR review | copilot |

## Verifier le statut

Un doute sur quel provider est actif ? Une seule commande :

```bash
$ aiden provider status
```

## Budget tokens

Les tokens, ca chiffre vite. AIDEN vous permet de poser des gardes-fous par feature et par mode, avec une alerte avant le mur.

```yaml
budget:
  monthlyLimit: 500000
  alertAt: 80
  perFeature:
    flash: 5000
    standard: 20000
    enterprise: 100000
```

> **Information** : L'alerte se declenche a 80% du budget. AIDEN bloque les appels si le budget est depasse.

## Filtrage securite

Avant chaque appel provider, AIDEN passe le prompt au crible. Les secrets ne sortent jamais de votre machine — meme par accident.

AIDEN filtre automatiquement les donnees sensibles avant de les envoyer aux providers :

- Bearer tokens
- API keys (`sk-`, `ghp_`, `github_pat_`)
- Mots de passe
- Cles privees (PEM)
- Connection strings (postgres, mongodb, redis)
- AWS access keys

Le filtrage est actif par defaut. Pour le desactiver (usage local uniquement) :

```yaml
sanitize: false
```
