# Regles et Templates

> Vos conventions d'equipe existent dans la tete de votre lead dev. Probleme : les agents IA ne lisent pas dans les pensees. Solution : ecrivez-les en Markdown, AIDEN les injecte automatiquement.

**Ce que vous allez apprendre :**
- Ecrire et ajouter des regles d'entreprise
- Personnaliser les templates d'artefacts
- Comment les regles transforment concretement les prompts des agents

## Regles d'entreprise

Les regles sont des fichiers `.md` dans `.aiden/rules/`. Elles sont automatiquement injectees dans le contexte de chaque agent. Pas d'API a configurer, pas de plugin a installer : du Markdown, c'est tout.

### Ajouter une regle

```bash
$ aiden rules add coding-standards.md
Rule added: coding-standards.md
```

### Exemple de regle

```markdown
# Standards de code

## TypeScript
- Strict mode obligatoire
- Pas de `any` sans justification
- Interfaces pour tous les types publics

## Tests
- Couverture minimale : 80%
- Tests unitaires pour toute logique metier

## Git
- Conventional Commits
- PR : max 400 lignes
```

### Lister les regles

```bash
$ aiden rules list
  coding-standards (.aiden/rules/coding-standards.md)
  security-policy (.aiden/rules/security-policy.md)
```

## Injection des regles : avant / apres

Sans regles, un agent Dev recoit un prompt generique. Il produit du code qui fonctionne, mais qui ne respecte pas forcement vos conventions. Avec des regles, le prompt de l'agent contient explicitement vos standards, et le code genere les respecte des la premiere iteration.

**Sans regles** : l'agent Dev genere du code valide, mais utilise `any`, oublie les interfaces publiques, et cree des commits sans convention.

**Avec `coding-standards.md`** : l'agent Dev recoit vos standards dans son prompt sous `--- Enterprise Rules ---`. Il produit du TypeScript strict, avec des interfaces, et des commits conventionnels.

Les regles sont filtrees par agent pour eviter le bruit :
- **Dev** recoit : coding-standards, security-policy
- **QA** recoit : coding-standards, security-policy
- **Planner** recoit : coding-standards, architecture-principles
- Les regles custom sont toujours envoyees a tous les agents

La difference est immediate : au lieu d'esperer que l'IA devine vos conventions, vous les lui donnez noir sur blanc.

## Templates

AIDEN fournit 7 templates par defaut (brief, spec, story, review, change-summary, architecture-decision, quality-report).

### Personnaliser un template

```bash
$ aiden template customize spec
Template "spec" copied to .aiden/templates/ for customization.
```

Le template copie utilise Handlebars. Variables disponibles : `feature_name`, `provider`, `mode`, `date`, etc.

Vous voulez que chaque spec generee contienne une section "Impact RGPD" ? Modifiez le template, et tous les agents qui produisent des specs l'incluront automatiquement.
