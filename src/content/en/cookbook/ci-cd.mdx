# CI/CD Integration

> AIDEN runs on the developer's machine, not in CI. But CI can verify that artifacts exist, block PRs without review, and collect metrics. Here's how.

**What you'll learn:**
- Validate AIDEN artifacts in CI
- Automate AIDEN checks in pipelines
- Integrate AIDEN metrics into your dashboards

## Principle

AIDEN runs **on the developer's workstation**, not in CI. But CI can:
1. **Verify** that AIDEN artifacts exist and are consistent
2. **Block** PRs without artifacts (if your policy requires it)
3. **Collect** metrics for the team dashboard
4. **Execute** non-interactive flashes for automated fixes

## GitHub Actions

### Verify AIDEN Artifacts on a PR

```yaml
# .github/workflows/aiden-check.yml
name: AIDEN Artefacts Check

on:
  pull_request:
    branches: [main, develop]

jobs:
  aiden-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check AIDEN initialized
        run: |
          if [ ! -f .aiden/aiden.config.yaml ]; then
            echo "::warning::AIDEN not initialized in this project"
            exit 0
          fi

      - name: Check feature artefacts
        run: |
          if [ -d .aiden/features ]; then
            for feature_dir in .aiden/features/*/; do
              if [ -d "$feature_dir" ]; then
                feature=$(basename "$feature_dir")
                echo "Checking feature: $feature"

                # In standard+ mode, verify brief and spec
                if [ -f "$feature_dir/spec.md" ] && [ ! -f "$feature_dir/review.md" ]; then
                  echo "::warning::Feature $feature has spec but no QA review"
                fi
              fi
            done
          fi

      - name: Check decisions trail
        run: |
          if [ -f .aiden/memory/decisions.jsonl ]; then
            echo "Decisions logged: $(wc -l < .aiden/memory/decisions.jsonl)"
          else
            echo "::warning::No decisions log found"
          fi
```

### Collect AIDEN Metrics

```yaml
# .github/workflows/aiden-metrics.yml
name: AIDEN Metrics

on:
  push:
    branches: [main]

jobs:
  metrics:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Extract AIDEN metrics
        run: |
          if [ -f .aiden/memory/metrics.jsonl ]; then
            echo "## AIDEN Metrics" >> $GITHUB_STEP_SUMMARY

            total_tokens=$(cat .aiden/memory/metrics.jsonl | \
              python3 -c "import sys,json; print(sum(json.loads(l)['tokensUsed'] for l in sys.stdin))")
            total_cost=$(cat .aiden/memory/metrics.jsonl | \
              python3 -c "import sys,json; print(f'{sum(json.loads(l).get(\"cost\",0) for l in sys.stdin):.2f}')")
            features=$(cat .aiden/memory/metrics.jsonl | \
              python3 -c "import sys,json; print(len(set(json.loads(l)['feature'] for l in sys.stdin)))")

            echo "| Metric | Value |" >> $GITHUB_STEP_SUMMARY
            echo "|----------|--------|" >> $GITHUB_STEP_SUMMARY
            echo "| Features | $features |" >> $GITHUB_STEP_SUMMARY
            echo "| Tokens | $total_tokens |" >> $GITHUB_STEP_SUMMARY
            echo "| Cost | \$$total_cost |" >> $GITHUB_STEP_SUMMARY
          fi
```

## GitLab CI

```yaml
# .gitlab-ci.yml
aiden-check:
  stage: validate
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
  script:
    - |
      if [ ! -f .aiden/aiden.config.yaml ]; then
        echo "AIDEN not initialized — skipping"
        exit 0
      fi

      for dir in .aiden/features/*/; do
        [ -d "$dir" ] || continue
        feature=$(basename "$dir")
        echo "Feature: $feature"
        ls -la "$dir"
      done

      if [ -f .aiden/memory/decisions.jsonl ]; then
        echo "Decisions: $(wc -l < .aiden/memory/decisions.jsonl)"
      fi
```

## Non-Interactive Mode

For automated tasks (lint fixes, dependency updates), AIDEN can run in CI:

```bash
$ aiden flash "Fix lint errors" --non-interactive --auto-approve
```

The `--non-interactive` flag disables prompts. The `--auto-approve` flag skips enterprise approvals.

**Warning**: reserve non-interactive mode for low-risk tasks (flash only). Standard and enterprise modes benefit from human validation.

## Pre-Commit Hook

Verify that artifacts are committed with the code:

```bash
#!/bin/sh
# .git/hooks/pre-commit

if git diff --cached --name-only | grep -q "^\.aiden/features/"; then
  if ! git diff --cached --name-only | grep -q "decisions.jsonl"; then
    echo "WARNING: Feature artifacts modified without decisions log."
    echo "Did you use AIDEN to generate these artifacts?"
  fi
fi
```

## Dashboards

### Export for Grafana / PowerBI

```bash
$ aiden metrics --export csv > aiden-metrics.csv
# Columns: timestamp, feature, agent, provider, tokensUsed, durationMs, cost
```

### Key Metrics for the Dashboard

| Metric | Source | Calculation |
|----------|--------|--------|
| Features/week | metrics.jsonl | Distinct features per week |
| Tokens/month | metrics.jsonl | Sum tokensUsed per month |
| Cost/month | metrics.jsonl | Sum cost per month |
| Most used provider | metrics.jsonl | Count per provider |
| Circuit breaker rate | decisions.jsonl | circuit_breaker / total |
| QA pass rate | decisions.jsonl | review=passed / total reviews |
