# Artifacts Reference

Every agent leaves a trace. Here are the 7 types of artifacts you'll find in .aiden/features/ — and what they contain.

---

## Common Format

Each artifact is a Markdown file with YAML frontmatter. This uniform format enables tracking, versioning and automatic injection into prompts:

```yaml
---
type: spec
feature: export-csv
version: 1
created: 2026-03-26
author: planner
status: draft
provider: claude-code
tokensUsed: 3421
mode: standard
---
```

Possible statuses: `draft`, `review`, `approved`, `implemented`.

---

## The 7 Artifacts

### brief.md

Generated by the **Analyst**. Summarizes the request, context, objective, constraints and risks. This is the starting point for any analysis — subsequent agents rely on it.

### spec.md

Generated by the **Planner**. Lightweight specification (1-2 pages) with requirements and acceptance criteria. Serves as a contract between planning and development.

### story.md (in stories/)

Generated by the **Planner**. Numbered user stories (001-*, 002-*) with acceptance criteria. Each story is an independent work unit for the Dev agent.

### review.md

Generated by **QA**. Code review result with score, issues and suggestions. If the score is insufficient, the workflow automatically re-runs the Dev with feedback.

### change-summary.md

Generated by **Dev** (flash mode). Summary of changes made — modified files, generated tests, decisions taken.

### architecture-decision.md

Generated by the **Planner** (enterprise mode). ADR: context, decision, alternatives, consequences. Records architectural choices for future reference.

### quality-report.md

Generated by **QA** (enterprise mode). Comprehensive report with metrics, security and recommendations. Includes explicit verification of each acceptance criterion.

---

## Location

All artifacts are in `.aiden/features/<feature-slug>/`.
