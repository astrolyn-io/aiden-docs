# Troubleshooting

> Not working? Don't panic. 90% of problems are solved in under a minute. Find yours below.

---

## Common problems

### "command not found: aiden"

**Symptom**: The terminal doesn't recognize the `aiden` command.

**Cause**: The binary isn't in the PATH.

**Solution**: Check the path and add it to your PATH:
```bash
$ pnpm bin -g
```

---

### "No AIDEN config found"

**Symptom**: AIDEN refuses to start and asks for a configuration.

**Cause**: The project hasn't been initialized.

**Solution**: Run `aiden init` at the root of your project.

---

### "No provider available"

**Symptom**: AIDEN can't find any provider to execute the task.

**Cause**: No CLI provider is installed or accessible.

**Solution**: Verify the provider is installed and responding:
```bash
$ claude --version   # or gemini --version
$ aiden provider status
```

---

### "Budget exceeded"

**Symptom**: AIDEN blocks execution and reports a budget overrun.

**Cause**: The feature's token budget has been exceeded.

**Solution**: Two options:
- Increase the budget in `aiden.config.yaml`
- Switch to flash mode (smaller but sufficient budget)

---

### Circuit breaker triggered

**Symptom**: The following message appears:
```
AIDEN Lead > Circuit breaker OPEN for dev: escalating to human
```

**Cause**: An agent failed 3 times in a row. AIDEN stops and escalates.

**Solution**: Check the error in the logs, fix the context, then resume:
```bash
$ aiden resume <slug>
```

---

### Feature blocked

**Symptom**: The feature shows as `blocked` in the status:
```bash
$ aiden status
my-feature [blocked] (standard) - ...
```

**Cause**: A quality gate failed or an approval is missing.

**Solution**: To unblock and resume:
```bash
$ aiden resume my-feature
Resuming feature: my-feature
```

To abandon permanently:
```bash
$ aiden abort my-feature
Aborting feature: my-feature
```

---

### Artifacts are not generated

**Symptom**: The workflow completes but no files appear.

**Cause**: The output directory doesn't exist or the provider isn't responding.

**Solution**: Verify that `.aiden/features/` exists and that `aiden provider status` shows an active provider.

---

## Logs and debug

When quick fixes aren't enough, the logs tell you exactly what happened.

Decisions are in `.aiden/memory/decisions.jsonl` (JSONL, one line per decision).
Metrics are in `.aiden/memory/metrics.jsonl`.

```bash
$ aiden metrics
export-csv: 3000 tokens, $0.0090, 2 operations
```

> **Warning**: Never manually edit `decisions.jsonl`. It's an append-only log.
