# Gemini CLI

1 million context tokens. 3 times cheaper than Claude for code generation. Gemini is the ideal partner for the Dev agent — and here's how to plug it in.

> **What you'll learn**
> - Install Gemini CLI (Google's AI coding assistant)
> - Configure Google authentication
> - Use Gemini as an AIDEN provider
> - Connect Gemini via direct API (alternative)
> - Resolve common issues

## Prerequisites

Before we get started, make sure you have:

- **Node.js 20+** — verify with `node --version`
- **Google account** with access to the Gemini API (Google AI Studio or Google Cloud)
- AIDEN installed and initialized (`aiden init`)

Nothing more. Let's go.

## Installation

Gemini CLI is Google's AI assistant for development. A simple global `npm install` and you're done:

```bash
$ npm install -g @google/gemini-cli
$ gemini --version
```

> **Note**: The package is called `@google/gemini-cli`. Don't confuse it with `@anthropic-ai/claude-code` which is Claude's CLI.

Verify the command is accessible:

```bash
$ gemini --help
```

If the command is not found, it's likely a PATH issue. Add the global npm directory:

```bash
$ npm bin -g
# Add this path to your PATH
```

## Authentication

Two methods depending on your context. The first is the simplest for local dev, the second is perfect for CI.

### Option 1 — Google OAuth (recommended for local development)

```bash
$ gemini auth login
```

This command opens your browser for Google OAuth authentication. Once connected, credentials are stored locally. No API key to manage — it's the smoothest method for daily use.

Verify the authentication:

```bash
$ gemini auth status
Logged in as: your.email@gmail.com
```

### Option 2 — Google AI Studio API Key

Generate a key on [Google AI Studio](https://aistudio.google.com/apikey), then export it:

```bash
$ export GOOGLE_API_KEY=AIzaSy...
```

To make it permanent, add the line to your `~/.bashrc` or `~/.zshrc`:

```bash
$ echo 'export GOOGLE_API_KEY=AIzaSy...' >> ~/.zshrc
$ source ~/.zshrc
```

> **Tip**: Prefer OAuth for local development to avoid storing keys in your shell. Reserve the API key for CI/CD environments where nobody can click on a browser.

## AIDEN Configuration

Now that we have a working Gemini, let's plug it into AIDEN. Add this to your `.aiden/aiden.config.yaml`:

```yaml
provider: gemini

providers:
  gemini:
    enabled: true
    command: gemini
    model: gemini-2.5-pro
    maxTokens: 1000000
    costPer1k: 0.001
```

This is the simplest configuration: Gemini does everything. But AIDEN's real power is multi-provider. You can keep Claude Code for analysis and review, and let Gemini do what it does best — generate code:

```yaml
provider: claude-code

providers:
  claude-code:
    enabled: true
    command: claude
    model: claude-sonnet-4-6
    maxTokens: 100000
    costPer1k: 0.003
  gemini:
    enabled: true
    command: gemini
    model: gemini-2.5-pro
    maxTokens: 1000000
    costPer1k: 0.001

routing:
  strategy: auto
  fallback: [claude-code, gemini]
```

In `auto` mode, AIDEN automatically routes the Dev agent to Gemini for code generation. Each agent uses the provider best suited to its task — without you having to think about it.

## First Usage

Set Gemini as the active provider and launch a feature to verify everything works:

```bash
$ aiden config set provider gemini
Set provider = gemini

$ aiden provider status
Provider status:
  gemini: configured
  claude-code: not found

$ aiden flash "Add CSV export to the reports module"
```

A quick `aiden provider status` before launching a long workflow avoids unpleasant surprises:

```bash
$ aiden provider status
```

## Alternative: Direct API

Don't want to install the CLI? Running in CI/CD or behind an LLM proxy? AIDEN's HTTP adapter is made for that.

Google exposes an OpenAI-compatible endpoint for Gemini, which means AIDEN can call it directly via `fetch()`:

```yaml
providers:
  gemini-api:
    enabled: true
    baseUrl: https://generativelanguage.googleapis.com/v1beta/openai
    apiKey: AIzaSy...
    model: gemini-2.5-pro
    maxTokens: 1000000
    costPer1k: 0.001
    command: ""
```

When `baseUrl` is set, AIDEN uses the HTTP adapter instead of the CLI subprocess — no `gemini` binary required. The API key is sent in the `Authorization: Bearer` header.

> **Note**: This approach also works via LiteLLM or Cloudflare AI Gateway if you want to centralize your LLM routing. Check the [Providers](/en/guides/providers) page for more details.

## Strengths and Use Cases

Why Gemini in an AIDEN pipeline? Here's what makes it interesting:

| Characteristic | Detail |
|---|---|
| **Context** | 1,000,000 tokens — ideal for large multi-file projects |
| **Code generation** | Excellent at structured code generation and refactoring |
| **Cost** | ~3x cheaper than Claude for code generation |
| **Speed** | Gemini 2.0 Flash: very fast for simple tasks |

In summary: one million context tokens for a third of the price. For pure code generation, that's hard to beat.

### Automatic Routing

In `auto` mode, the **Dev** agent (`code_gen`) is routed to Gemini by default. Analysis and review agents stay on Claude Code, which excels at deep understanding:

| Agent | TaskType | Provider (auto) |
|---|---|---|
| Analyst | analysis | claude-code |
| Planner | planning | claude-code |
| **Dev** | **code_gen** | **gemini** |
| QA | code_review | claude-code |
| DocOps | documentation | claude-code |

The best of both worlds: Claude thinks, Gemini codes.

### Available Models

| Model | Recommended usage |
|---|---|
| `gemini-2.5-pro` | Complex code generation and analysis |
| `gemini-2.0-flash` | Simple tasks, fast iterations, flash mode |
| `gemini-1.5-pro` | Stable alternative if 2.5 is unavailable |

To change models, just one line to modify:

```yaml
providers:
  gemini:
    model: gemini-2.0-flash
```

## Troubleshooting

The most common issues and how to fix them in 30 seconds.

### "command not found: gemini"

The binary is not in the PATH. Check the installation:

```bash
$ npm list -g @google/gemini-cli
$ npm bin -g
```

Add the path returned by `npm bin -g` to your PATH.

### "Authentication required" or "UNAUTHENTICATED"

The OAuth authentication has expired or the API key is missing. Reconnect:

```bash
$ gemini auth login
```

Or verify that the environment variable is set:

```bash
$ echo $GOOGLE_API_KEY
```

### "Model not found" or "404"

The specified model doesn't exist or isn't available in your region. Fall back to `gemini-2.0-flash` which is available everywhere:

```yaml
providers:
  gemini:
    model: gemini-2.0-flash
```

### Timeout on Large Features

AIDEN's default timeout is 120 seconds. With the 1M token context window, very long prompts can exceed this limit. Increase the timeout in `.aiden/aiden.config.yaml`:

```yaml
providers:
  gemini:
    timeout: 300
```

### "No provider available" After Configuration

Verify that the provider is enabled and accessible:

```bash
$ aiden provider status
$ gemini --version
```

If `gemini --version` fails, reinstall the CLI.

### API Quota Limits

Google AI Studio has requests per minute (RPM) limits. If you hit them, AIDEN will show a `ProviderError`. Three solutions:

- Switch to a Google Cloud project with billing enabled (higher quotas)
- Add a delay between features in your workflow
- Use LiteLLM as a proxy with rate limiting management
