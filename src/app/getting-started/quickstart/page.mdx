# Quickstart

> **Ce que vous allez apprendre**
> - Creer un projet AIDEN de zero en 5 minutes
> - Lancer un workflow flash et voir le resultat
> - Comprendre ce que AIDEN a fait

## Etape 1 : Creer un projet

```bash
$ mkdir mon-api && cd mon-api
$ npm init -y
$ npm install express
```

Creez un fichier `index.js` minimal :

```js
const express = require('express');
const app = express();
app.get('/', (req, res) => res.json({ status: 'ok' }));
module.exports = app;
```

## Etape 2 : Initialiser AIDEN

```bash
$ aiden init
AIDEN initialized in .aiden/
  Config: .aiden/aiden.config.yaml
  Rules:  .aiden/rules/coding-standards.md
  Memory: .aiden/memory/
  Stack:  node, express
  PM:     npm
```

AIDEN a detecte que c'est un projet Node.js avec Express.

## Etape 3 : Lancer un flash

```bash
$ aiden flash "Ajouter un endpoint GET /health retournant {status: healthy}"
```

Output :

```
Starting flash workflow for: Ajouter un endpoint GET /health...
  Phase: implementation, Agent: dev, Provider: claude-code
  Dev: Implementing feature...
  Dev: Writing tests...
  Feature completed: ajouter-un-endpoint-get-health
  Tokens: 1250 | Duration: 12s
```

## Etape 4 : Verifier le resultat

```bash
$ aiden status
ajouter-un-endpoint-get-health [completed] (flash)
  Tokens: 1250 | Created: 2026-03-27
```

Explorez les artefacts :

```bash
$ ls .aiden/features/ajouter-un-endpoint-get-health/
change-summary.md
```

Le `change-summary.md` contient un resume de ce que le Dev a fait : fichiers modifies, tests crees, decisions prises.

## Etape 5 : Verifier le code

Le Dev a modifie `index.js` pour ajouter la route et cree un fichier de test. Verifiez :

```bash
$ cat index.js
# Devrait contenir: app.get('/health', ...)

$ ls *.test.* *.spec.*
# Un fichier de test devrait exister
```

## Etape 6 : Voir les metriques

```bash
$ aiden metrics
Features: 1 completed
Tokens:   1250 used
Cost:     $0.004
```

## Ce que AIDEN a fait

1. **Detection** : Le Lead a analyse la demande et choisi le mode flash (simple, < 3 fichiers)
2. **Implementation** : Le Dev a lu le code existant, ajoute la route `/health`, ecrit un test
3. **Tracabilite** : La decision est logguee dans `.aiden/memory/decisions.jsonl`, le contexte est mis a jour

## Checklist de reussite

- `.aiden/` existe avec la config et la memoire
- `aiden status` montre la feature completee
- Un `change-summary.md` a ete genere
- Le code contient la route `/health`

## Si ca ne marche pas

- **"AIDEN not initialized"** — Lancez `aiden init` d'abord
- **"No provider available"** — Verifiez que `claude --version` fonctionne
- **Le provider timeout** — La premiere requete peut etre lente, reessayez
- **Le code n'est pas ce que vous attendiez** — Essayez `aiden standard` pour plus de controle (analyse + planning + review)
