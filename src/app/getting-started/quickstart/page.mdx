# Quickstart

> **Ce que vous allez apprendre**
> - Créer un projet AIDEN de zéro en 5 minutes
> - Lancer un workflow flash et voir le résultat
> - Comprendre ce que AIDEN a fait

## Étape 1 : Créer un projet

```bash
$ mkdir mon-api && cd mon-api
$ npm init -y
$ npm install express
```

Créez un fichier `index.js` minimal :

```js
const express = require('express');
const app = express();
app.get('/', (req, res) => res.json({ status: 'ok' }));
module.exports = app;
```

## Étape 2 : Initialiser AIDEN

```bash
$ aiden init
AIDEN initialized in .aiden/
  Config: .aiden/aiden.config.yaml
  Rules:  .aiden/rules/coding-standards.md
  Memory: .aiden/memory/
  Stack:  node, express
  PM:     npm
```

AIDEN a détecté que c'est un projet Node.js avec Express.

## Étape 3 : Lancer un flash

```bash
$ aiden flash "Ajouter un endpoint GET /health retournant {status: healthy}"
```

Output :

```
Starting flash workflow for: Ajouter un endpoint GET /health...
  Phase: implementation, Agent: dev, Provider: claude-code
  Dev: Implementing feature...
  Dev: Writing tests...
  Feature completed: ajouter-un-endpoint-get-health
  Tokens: 1250 | Duration: 12s
```

## Étape 4 : Vérifier le résultat

```bash
$ aiden status
ajouter-un-endpoint-get-health [completed] (flash)
  Tokens: 1250 | Created: 2026-03-27
```

Explorez les artefacts :

```bash
$ ls .aiden/features/ajouter-un-endpoint-get-health/
change-summary.md
```

Le `change-summary.md` contient un résumé de ce que le Dev a fait : fichiers modifiés, tests créés, décisions prises.

## Étape 5 : Vérifier le code

Le Dev a modifié `index.js` pour ajouter la route et créé un fichier de test. Vérifiez :

```bash
$ cat index.js
# Devrait contenir: app.get('/health', ...)

$ ls *.test.* *.spec.*
# Un fichier de test devrait exister
```

## Étape 6 : Voir les métriques

```bash
$ aiden metrics
Features: 1 completed
Tokens:   1250 used
Cost:     $0.004
```

## Ce que AIDEN a fait

1. **Détection** : Le Lead a analysé la demande et choisi le mode flash (simple, < 3 fichiers)
2. **Implémentation** : Le Dev a lu le code existant, ajouté la route `/health`, écrit un test
3. **Traçabilité** : La décision est logguée dans `.aiden/memory/decisions.jsonl`, le contexte est mis à jour

## Checklist de réussite

- `.aiden/` existe avec la config et la mémoire
- `aiden status` montre la feature complétée
- Un `change-summary.md` a été généré
- Le code contient la route `/health`

## Si ça ne marche pas

- **"AIDEN not initialized"** — Lancez `aiden init` d'abord
- **"No provider available"** — Vérifiez que `claude --version` fonctionne
- **Le provider timeout** — La première requête peut être lente, réessayez
- **Le code n'est pas ce que vous attendiez** — Essayez `aiden standard` pour plus de contrôle (analyse + planning + review)
