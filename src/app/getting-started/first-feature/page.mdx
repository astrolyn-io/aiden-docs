# Premiere feature en mode standard

> **Ce que vous allez apprendre**
> - Le workflow standard : Analyst, Planner, Dev, QA
> - Les artefacts generes a chaque phase
> - Suivre l'avancement d'une feature

## Scenario : ajouter GET /health

```bash
$ cta standard "Ajouter un endpoint GET /health"
Starting standard workflow...
```

## Phase 1 — Decouverte (Analyst)

L'Analyst scanne le code et pose 3-5 questions :

```
CTA Analyst > Question 1/3 : Format de reponse JSON ?
CTA Analyst > Question 2/3 : Verifier la connexion DB ?
CTA Analyst > Question 3/3 : Endpoint public ou protege ?
```

Artefact : `.cta/features/ajouter-endpoint-get-health/brief.md`

## Phase 2 — Planification (Planner)

Le Planner cree spec et stories :

```
CTA Planner > Spec creee (1 page)
CTA Planner > 2 stories : endpoint + tests
```

Artefacts : `spec.md`, `stories/001-*.md`, `stories/002-*.md`

## Phase 3 — Implementation (Dev)

Le Dev implemente chaque story avec tests et self-review.

## Phase 4 — Validation (QA)

Le QA revoit le code, verifie les criteres, scan securite.

Artefact : `review.md`

## Resultat

```bash
$ cta status ajouter-endpoint-get-health
Feature: Ajouter un endpoint GET /health
  State: completed
  Mode: standard
  Tokens: 450
```

> **Information** : Tous les artefacts dans `.cta/features/<slug>/` servent de trace pour reprendre le contexte.
