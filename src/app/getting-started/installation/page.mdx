# Installation

> **Ce que vous allez apprendre**
> - Installer AIDEN et ses prérequis
> - Initialiser un projet avec `aiden init`
> - Vérifier que tout fonctionne

## Prérequis

- **Node.js 20+** — vérifiez avec `node --version`
- **pnpm** — installez avec `npm install -g pnpm`
- Au moins un provider IA : Claude Code, Gemini CLI, OpenCode ou GitHub Copilot (ou un proxy API type LiteLLM)

## Installer AIDEN

```bash
$ pnpm add -g @astrolyn/aiden
```

Vérifiez :

```bash
$ aiden --version
0.7.0
```

> **Astuce** : Si `command not found`, ajoutez le dossier global pnpm à votre PATH : `export PATH="$(pnpm store path)/../../.pnpm:$PATH"`

## Initialiser un projet

Dans le répertoire de votre projet :

```bash
$ aiden init
AIDEN initialized in .aiden/
  Config: .aiden/aiden.config.yaml
  Rules:  .aiden/rules/coding-standards.md
  Memory: .aiden/memory/
  Stack:  node, typescript, express
  Arch:   docker
  CI:     github-actions
  PM:     pnpm
```

AIDEN scanne automatiquement votre projet et détecte :
- **Tech stack** : Node.js, TypeScript, React, Express, NestJS, Vue, Angular, Python, Go, Rust...
- **Architecture** : Docker, Docker Compose, Kubernetes
- **CI/CD** : GitHub Actions, GitLab CI, Jenkins, CircleCI
- **Package manager** : pnpm, yarn, npm, bun
- **Databases** : PostgreSQL, MySQL, Redis, MongoDB (depuis les deps et docker-compose)
- **Monorepo** : Turborepo, Nx, Lerna, pnpm workspaces

Ces informations sont injectées dans le contexte de chaque agent pour des résultats plus pertinents.

Structure créée :

```
.aiden/
  aiden.config.yaml    # Configuration projet
  memory/            # Contexte, décisions, métriques
  rules/             # Règles entreprise
  templates/         # Templates personnalisés
  features/          # Features en cours
```

## Configurer le provider

Par défaut, AIDEN utilise Claude Code. Pour changer :

```bash
$ aiden config set provider gemini
Set provider = gemini
```

Providers disponibles : `claude-code`, `gemini`, `copilot`.

Vérifiez que votre provider est accessible :

```bash
$ aiden provider status
Provider status:
  claude-code: configured
  gemini: not found
  copilot: configured
```

## Premier test

```bash
$ aiden flash "Ajouter hello.ts" --dry-run
[DRY RUN] Flash workflow: "Ajouter hello.ts"
Mode: flash | Agents: lead, dev
```

Le `--dry-run` montre ce qui serait fait sans appeler le provider.

## Mode slash (Claude Code)

Si vous utilisez Claude Code, AIDEN est aussi disponible en slash commands sans installer le CLI :

- `/aiden-init` — initialiser AIDEN
- `/aiden-flash` — mode rapide
- `/aiden-standard` — mode complet
- `/aiden-enterprise` — mode enterprise

Les deux modes (CLI et slash) écrivent dans les mêmes fichiers `.aiden/`.

## Troubleshooting

**`command not found: aiden`** — Assurez-vous que le dossier global pnpm est dans votre PATH.

**`AIDEN not initialized`** — Lancez `aiden init` dans le répertoire du projet.

**`No provider available`** — Installez Claude Code (`npm install -g @anthropic-ai/claude-code`) ou un autre provider.

**Timeout sur le provider** — Le timeout par défaut est 120s. Si votre LLM est lent, augmentez-le dans `.aiden/aiden.config.yaml`.
