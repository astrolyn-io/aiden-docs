# Règles et Templates

> **Ce que vous allez apprendre**
> - Écrire et ajouter des règles d'entreprise
> - Personnaliser les templates d'artefacts
> - Comment les règles sont injectées dans les agents

## Règles d'entreprise

Les règles sont des fichiers `.md` dans `.aiden/rules/`. Elles sont automatiquement injectées dans le contexte de chaque agent.

### Ajouter une règle

```bash
$ aiden rules add coding-standards.md
Rule added: coding-standards.md
```

### Exemple de règle

```markdown
# Standards de code

## TypeScript
- Strict mode obligatoire
- Pas de `any` sans justification
- Interfaces pour tous les types publics

## Tests
- Couverture minimale : 80%
- Tests unitaires pour toute logique metier

## Git
- Conventional Commits
- PR : max 400 lignes
```

### Lister les règles

```bash
$ aiden rules list
  coding-standards (.aiden/rules/coding-standards.md)
  security-policy (.aiden/rules/security-policy.md)
```

## Injection des règles

Les règles sont filtrées par agent :
- **Dev** reçoit : coding-standards, security-policy
- **QA** reçoit : coding-standards, security-policy
- **Planner** reçoit : coding-standards, architecture-principles
- Les règles custom sont toujours envoyées à tous les agents

## Templates

AIDEN fournit 7 templates par défaut (brief, spec, story, review, change-summary, architecture-decision, quality-report).

### Personnaliser un template

```bash
$ aiden template customize spec
Template "spec" copied to .aiden/templates/ for customization.
```

Le template copié utilise Handlebars. Variables disponibles : `feature_name`, `provider`, `mode`, `date`, etc.
