# Troubleshooting

> **Ce que vous allez apprendre**
> - Resoudre les problemes courants
> - Lire les logs AIDEN
> - Resetrer une feature bloquee

## Problemes courants

### "command not found: cta"

Le binaire n'est pas dans le PATH. Verifiez :
```bash
$ pnpm bin -g
```
Ajoutez ce chemin a votre PATH.

### "No AIDEN config found"

Vous n'avez pas initialise AIDEN. Lancez `aiden init`.

### "No provider available"

Aucun provider n'est accessible. Verifiez :
```bash
$ claude --version   # ou gemini --version
$ aiden provider status
```

### "Budget exceeded"

Le budget tokens de la feature est depasse. Options :
- Augmenter le budget dans `aiden.config.yaml`
- Passer en mode flash (budget plus petit mais suffisant)

### Circuit breaker declenche

Un agent a echoue 3 fois. AIDEN arrete et escalade :
```
AIDEN Lead > Circuit breaker OPEN for dev: escalating to human
```
Solution : verifiez l'erreur, corrigez le contexte, puis `aiden resume <slug>`.

### Feature bloquee

```bash
$ aiden status
ma-feature [blocked] (standard) - ...
```

Pour debloquer :
```bash
$ aiden resume ma-feature
Resuming feature: ma-feature
```

Pour abandonner :
```bash
$ aiden abort ma-feature
Aborting feature: ma-feature
```

### Les artefacts ne sont pas generes

Verifiez que le dossier `.aiden/features/` existe et que le provider repond.

## Logs

Les decisions sont dans `.aiden/memory/decisions.jsonl` (JSONL, une ligne par decision).
Les metriques sont dans `.aiden/memory/metrics.jsonl`.

```bash
$ aiden metrics
export-csv: 3000 tokens, $0.0090, 2 operations
```

> **Danger** : Ne modifiez jamais `decisions.jsonl` manuellement. C'est un log append-only.
