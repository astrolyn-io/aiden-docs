# Troubleshooting

> **Ce que vous allez apprendre**
> - Résoudre les problèmes courants
> - Lire les logs AIDEN
> - Réinitialiser une feature bloquée

## Problèmes courants

### "command not found: aiden"

Le binaire n'est pas dans le PATH. Vérifiez :
```bash
$ pnpm bin -g
```
Ajoutez ce chemin à votre PATH.

### "No AIDEN config found"

Vous n'avez pas initialisé AIDEN. Lancez `aiden init`.

### "No provider available"

Aucun provider n'est accessible. Vérifiez :
```bash
$ claude --version   # ou gemini --version
$ aiden provider status
```

### "Budget exceeded"

Le budget tokens de la feature est dépassé. Options :
- Augmenter le budget dans `aiden.config.yaml`
- Passer en mode flash (budget plus petit mais suffisant)

### Circuit breaker déclenché

Un agent a échoué 3 fois. AIDEN arrête et escalade :
```
AIDEN Lead > Circuit breaker OPEN for dev: escalating to human
```
Solution : vérifiez l'erreur, corrigez le contexte, puis `aiden resume <slug>`.

### Feature bloquée

```bash
$ aiden status
ma-feature [blocked] (standard) - ...
```

Pour débloquer :
```bash
$ aiden resume ma-feature
Resuming feature: ma-feature
```

Pour abandonner :
```bash
$ aiden abort ma-feature
Aborting feature: ma-feature
```

### Les artefacts ne sont pas générés

Vérifiez que le dossier `.aiden/features/` existe et que le provider répond.

## Logs

Les décisions sont dans `.aiden/memory/decisions.jsonl` (JSONL, une ligne par décision).
Les métriques sont dans `.aiden/memory/metrics.jsonl`.

```bash
$ aiden metrics
export-csv: 3000 tokens, $0.0090, 2 operations
```

> **Danger** : Ne modifiez jamais `decisions.jsonl` manuellement. C'est un log append-only.
