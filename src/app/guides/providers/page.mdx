# Providers

> **Ce que vous allez apprendre**
> - Configurer Claude Code, Gemini et Copilot

## Providers supportes

CTA supporte 3 providers IA nativement :

| Provider | Commande | Forces |
|---|---|---|
| Claude Code | `claude` | Analyse, review, long contexte |
| Gemini CLI | `gemini` | Generation code, multi-fichier |
| GitHub Copilot | `gh copilot` | Completion, quick fix |

## Configuration

Dans `.cta/cta.config.yaml` :

```yaml
provider: claude-code
providers:
  claude-code:
    enabled: true
    command: claude
    model: claude-sonnet-4-6
    maxTokens: 100000
    costPer1k: 0.003
  gemini:
    enabled: true
    command: gemini
    model: gemini-2.5-pro
    maxTokens: 1000000
    costPer1k: 0.001
```

## Routage

4 strategies disponibles :

- **prefer** (defaut) : utilise le provider prefere, fallback si indisponible
- **auto** : choisit le meilleur provider selon la tache
- **fixed** : un seul provider toujours
- **cost-optimized** : minimise les couts

```yaml
routing:
  strategy: auto
  fallback: [gemini, claude-code, copilot]
```

```yaml

  monthlyLimit: 500000
  alertAt: 80
  perFeature:
    flash: 5000
    standard: 20000
    enterprise: 100000
```

> **Information** : L'alerte se declenche a 80% du budget. CTA bloque les appels si le budget est depasse.

## Routage dans l'orchestrateur

Le router est directement branche dans l'orchestrateur. Chaque agent recoit son provider via `router.route(taskType)` — pas un provider unique pour tout le workflow. En mode `auto`, l'Analyst utilise Claude Code (meilleur en analyse) tandis que le Dev utilise Gemini (meilleur en generation de code).

```yaml
routing:
  strategy: auto
  fallback: [claude-code, gemini, copilot]
```

Si un provider tombe pendant un workflow, le fallback est automatique — l'agent suivant recoit un provider fonctionnel.

## Communication avec les providers

CTA envoie les prompts aux providers via **stdin pipe** (pas comme argument de commande). Cela permet d'envoyer des prompts de taille illimitee sans etre contraint par les limites shell.

- **Timeout** : 120 secondes par defaut. Si un provider ne repond pas dans ce delai, `ProviderTimeoutError` est leve et le circuit breaker est notifie.
- **Format de review** : CTA demande aux providers de repondre dans un format structure (`SCORE: N / PASSED: yes|no / ISSUES: ...`). Si le provider ne respecte pas le format, un fallback heuristique est utilise.
- **Module partage** : Les 3 adapters utilisent `src/providers/subprocess.ts` — pas de duplication de code entre providers.
