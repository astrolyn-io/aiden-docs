## Les 5 niveaux

Chaque niveau surcharge le précédent :

1. **Défauts AIDEN** (built-in)
2. **Enterprise** (`~/.aiden/enterprise.config.yaml`)
3. **Projet** (`.aiden/aiden.config.yaml`)
4. **Feature** (`.aiden/features/<slug>/.override.yaml`)
5. **CLI flags** (`--mode`, `--provider`, etc.)

## Configuration minimale

```yaml
# .aiden/aiden.config.yaml
project: mon-projet
provider: claude-code
mode: standard
lang: fr
```

4 lignes suffisent pour démarrer.

## Options complètes

| Clé | Type | Défaut | Description |
|---|---|---|---|
| `project` | string | (requis) | Nom du projet |
| `provider` | string | `claude-code` | Provider par défaut |
| `mode` | enum | `standard` | flash, standard, enterprise |
| `lang` | string | `fr` | Langue des artefacts |
| `providers` | object | (built-in) | Config par provider |
| `routing` | object | prefer | Stratégie de routage |
| `budget` | object | (built-in) | Budget tokens |
| `hooks` | object | {} | Hooks d'événements |
| `rules` | array | [] | Liste des fichiers de règles |
| `pipeline` | object | (voir ci-dessous) | Configuration du pipeline inter-agents |

### Pipeline

```yaml
pipeline:
  maxOutputCharsPerAgent: 8000  # Troncature des outputs agents (defaut: 8000)
```

Cette option contrôle la quantité de texte d'un agent précédent qui est injectée dans le prompt du suivant. Avec un LLM à grand contexte (200k+), vous pouvez augmenter cette valeur. Réduisez-la si vous atteignez les limites de tokens.

## Configuration entreprise

```yaml
# ~/.aiden/enterprise.config.yaml
company: my-company
defaults:
  mode: standard
  lang: fr
  provider: claude-code
  require_approval: true
security:
  no_secrets_in_artefacts: true
  scan_dependencies: true
coding:
  test_coverage_min: 80
  commit_convention: conventional
```

## Modifier via CLI

```bash
$ aiden config set mode flash
Set mode = flash

$ aiden config show
{ "project": "mon-projet", "mode": "flash", ... }
```
