# Intégration CI/CD

> **Ce que vous allez apprendre**
> - Valider les artefacts AIDEN dans la CI
> - Automatiser les checks AIDEN dans les pipelines
> - Intégrer les métriques AIDEN dans vos dashboards

## Principe

AIDEN s'exécute **sur le poste du développeur**, pas dans la CI. Mais la CI peut :
1. **Vérifier** que les artefacts AIDEN existent et sont cohérents
2. **Bloquer** les PRs sans artefacts (si votre politique l'exige)
3. **Collecter** les métriques pour le dashboard d'équipe
4. **Exécuter** des flash non-interactifs pour les fix automatiques

## GitHub Actions

### Vérifier les artefacts AIDEN sur une PR

```yaml
# .github/workflows/aiden-check.yml
name: AIDEN Artefacts Check

on:
  pull_request:
    branches: [main, develop]

jobs:
  aiden-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check AIDEN initialized
        run: |
          if [ ! -f .aiden/aiden.config.yaml ]; then
            echo "::warning::AIDEN not initialized in this project"
            exit 0
          fi

      - name: Check feature artefacts
        run: |
          if [ -d .aiden/features ]; then
            for feature_dir in .aiden/features/*/; do
              if [ -d "$feature_dir" ]; then
                feature=$(basename "$feature_dir")
                echo "Checking feature: $feature"

                # En mode standard+, verifier brief et spec
                if [ -f "$feature_dir/spec.md" ] && [ ! -f "$feature_dir/review.md" ]; then
                  echo "::warning::Feature $feature has spec but no QA review"
                fi
              fi
            done
          fi

      - name: Check decisions trail
        run: |
          if [ -f .aiden/memory/decisions.jsonl ]; then
            echo "Decisions logged: $(wc -l < .aiden/memory/decisions.jsonl)"
          else
            echo "::warning::No decisions log found"
          fi
```

### Collecter les métriques AIDEN

```yaml
# .github/workflows/aiden-metrics.yml
name: AIDEN Metrics

on:
  push:
    branches: [main]

jobs:
  metrics:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Extract AIDEN metrics
        run: |
          if [ -f .aiden/memory/metrics.jsonl ]; then
            echo "## AIDEN Metrics" >> $GITHUB_STEP_SUMMARY

            total_tokens=$(cat .aiden/memory/metrics.jsonl | \
              python3 -c "import sys,json; print(sum(json.loads(l)['tokensUsed'] for l in sys.stdin))")
            total_cost=$(cat .aiden/memory/metrics.jsonl | \
              python3 -c "import sys,json; print(f'{sum(json.loads(l).get(\"cost\",0) for l in sys.stdin):.2f}')")
            features=$(cat .aiden/memory/metrics.jsonl | \
              python3 -c "import sys,json; print(len(set(json.loads(l)['feature'] for l in sys.stdin)))")

            echo "| Métrique | Valeur |" >> $GITHUB_STEP_SUMMARY
            echo "|----------|--------|" >> $GITHUB_STEP_SUMMARY
            echo "| Features | $features |" >> $GITHUB_STEP_SUMMARY
            echo "| Tokens | $total_tokens |" >> $GITHUB_STEP_SUMMARY
            echo "| Coût | \$$total_cost |" >> $GITHUB_STEP_SUMMARY
          fi
```

## GitLab CI

```yaml
# .gitlab-ci.yml
aiden-check:
  stage: validate
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
  script:
    - |
      if [ ! -f .aiden/aiden.config.yaml ]; then
        echo "AIDEN not initialized — skipping"
        exit 0
      fi

      for dir in .aiden/features/*/; do
        [ -d "$dir" ] || continue
        feature=$(basename "$dir")
        echo "Feature: $feature"
        ls -la "$dir"
      done

      if [ -f .aiden/memory/decisions.jsonl ]; then
        echo "Decisions: $(wc -l < .aiden/memory/decisions.jsonl)"
      fi
```

## Mode non-interactif

Pour les tâches automatisées (fix de lint, updates de deps), AIDEN peut tourner en CI :

```bash
$ aiden flash "Fix lint errors" --non-interactive --auto-approve
```

Le `--non-interactive` désactive les prompts. Le `--auto-approve` skip les approbations enterprise.

**Attention** : réservez le mode non-interactif aux tâches à faible risque (flash uniquement). Les modes standard et enterprise bénéficient de la validation humaine.

## Pre-commit hook

Vérifier que les artefacts sont commités avec le code :

```bash
#!/bin/sh
# .git/hooks/pre-commit

if git diff --cached --name-only | grep -q "^\.aiden/features/"; then
  if ! git diff --cached --name-only | grep -q "decisions.jsonl"; then
    echo "WARNING: Feature artefacts modifies sans decisions log."
    echo "Avez-vous utilise AIDEN pour generer ces artefacts ?"
  fi
fi
```

## Dashboards

### Export pour Grafana / PowerBI

```bash
$ aiden metrics --export csv > aiden-metrics.csv
# Colonnes: timestamp, feature, agent, provider, tokensUsed, durationMs, cost
```

### Métriques clés pour le dashboard

| Métrique | Source | Calcul |
|----------|--------|--------|
| Features/semaine | metrics.jsonl | Distinct features par semaine |
| Tokens/mois | metrics.jsonl | Sum tokensUsed par mois |
| Coût/mois | metrics.jsonl | Sum cost par mois |
| Provider le plus utilisé | metrics.jsonl | Count par provider |
| Taux circuit breaker | decisions.jsonl | circuit_breaker / total |
| Taux QA pass | decisions.jsonl | review=passed / total reviews |
