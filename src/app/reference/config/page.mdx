# Reference Configuration

> **Ce que vous allez apprendre**
> - Le schema complet de `cta.config.yaml`
> - Les valeurs par defaut
> - Quel niveau surcharge quoi

## Schema complet

```yaml
# .cta/cta.config.yaml
project: mon-projet              # (requis) Nom du projet
provider: claude-code            # Provider par defaut
mode: standard                   # flash | standard | enterprise
lang: fr                         # Langue des artefacts

providers:                       # Configuration par provider
  claude-code:
    enabled: true
    command: claude
    model: claude-sonnet-4-6
    maxTokens: 100000
    costPer1k: 0.003
  gemini:
    enabled: true
    command: gemini
    model: gemini-2.5-pro
    maxTokens: 1000000
    costPer1k: 0.001
  copilot:
    enabled: true
    command: "gh copilot"
    model: copilot
    maxTokens: 50000
    costPer1k: 0

routing:                         # Strategie de routage
  strategy: prefer               # auto | prefer | fixed | cost-optimized
  fallback: [gemini, claude-code, copilot]

budget:                          # Budget tokens
  monthlyLimit: 500000
  alertAt: 80                    # Pourcentage
  perFeature:
    flash: 5000
    standard: 20000
    enterprise: 100000

hooks: {}                        # Hooks par evenement
rules: []                        # Fichiers de regles

pipeline:                        # Configuration du pipeline inter-agents
  maxOutputCharsPerAgent: 8000   # Troncature outputs (defaut: 8000)
```

### Options pipeline

| Cle | Type | Defaut | Description |
|---|---|---|---|
| `pipeline.maxOutputCharsPerAgent` | number | 8000 | Nombre max de caracteres par output d'agent injecte dans le prompt du suivant. Augmentez pour les LLMs a grand contexte, reduisez si vous atteignez les limites de tokens. |

### Options routing

| Cle | Type | Defaut | Description |
|---|---|---|---|
| `routing.strategy` | string | `prefer` | Strategie de routage : `fixed` (un seul provider), `prefer` (prefere + fallback), `auto` (choix par type de tache), `cost-optimized` (minimise les couts). En mode `auto`, chaque agent recoit le provider optimal : Analyst → claude-code, Dev → gemini, etc. |
| `routing.fallback` | string[] | `[claude-code, gemini, copilot]` | Chaine de fallback si le provider prefere est indisponible. |

### Schema projet (context.yaml)

Le scan de `cta init` genere `.cta/memory/context.yaml` avec ce schema :

| Cle | Type | Description |
|---|---|---|
| `project.name` | string | Nom du projet (depuis le dossier) |
| `project.techStack` | string[] | Technologies detectees (node, typescript, react, nestjs, prisma, python...) |
| `project.architecture` | string[] | Infrastructure (docker, docker-compose, kubernetes) |
| `project.ci` | string[] | Systemes CI/CD (github-actions, gitlab-ci, jenkins, circleci) |
| `project.packageManager` | string? | Gestionnaire de paquets (pnpm, yarn, npm, bun) |
| `project.databases` | string[]? | Bases de donnees (postgres, mysql, redis, mongodb) |
| `project.monorepo` | string? | Outil monorepo (turborepo, nx, lerna, pnpm-workspaces) |

Ces donnees sont injectees dans le prompt de chaque agent via la section `--- Project Context ---`.

## Niveaux de surcharge

| Niveau | Fichier | Portee |
|---|---|---|
| 1. Defauts | (built-in) | Global |
| 2. Enterprise | `~/.cta/enterprise.config.yaml` | Tous les projets |
| 3. Projet | `.cta/cta.config.yaml` | Ce projet |
| 4. Feature | `.cta/features/<slug>/.override.yaml` | Cette feature |
| 5. CLI | `--mode`, `--provider` | Cette commande |

Chaque niveau ecrase les valeurs du precedent. Les objets sont fusionnes en profondeur, les tableaux sont remplaces.
