# Référence Configuration

> **Ce que vous allez apprendre**

> - Quel niveau surcharge quoi

## Schéma complet

```yaml
# .aiden/aiden.config.yaml
project: mon-projet              # (requis) Nom du projet
provider: claude-code            # Provider par defaut
mode: standard                   # flash | standard | enterprise
lang: fr                         # Langue des artefacts

providers:                       # Configuration par provider
  claude-code:
    enabled: true
    command: claude
    model: claude-sonnet-4-6
    maxTokens: 100000
    costPer1k: 0.003
  gemini:
    enabled: true
    command: gemini
    model: gemini-2.5-pro
    maxTokens: 1000000
    costPer1k: 0.001
  opencode:
    enabled: true
    command: opencode
    model: opencode
    maxTokens: 200000
    costPer1k: 0.002
  copilot:
    enabled: true
    command: gh
    model: copilot
    maxTokens: 50000
    costPer1k: 0
  # Provider API custom (proxy LLM)
  litellm:
    enabled: true
    baseUrl: http://localhost:4000/v1   # endpoint OpenAI-compatible
    # apiKey: sk-xxx                    # optionnel
    model: claude-sonnet-4-6
    maxTokens: 100000
    costPer1k: 0.003
    command: ""                         # pas de subprocess

routing:                         # Strategie de routage
  strategy: prefer               # auto | prefer | fixed | cost-optimized
  fallback: [claude-code, gemini, opencode, copilot]

budget:                          # Budget tokens
  monthlyLimit: 500000
  alertAt: 80                    # Pourcentage
  perFeature:
    flash: 5000
    standard: 20000
    enterprise: 100000

hooks: {}                        # Hooks par evenement
rules: []                        # Fichiers de regles

pipeline:                        # Configuration du pipeline inter-agents
  maxOutputCharsPerAgent: 8000   # Troncature outputs (defaut: 8000)

sanitize: true                   # Filtrage donnees sensibles (defaut: true)
```

### Options providers

| Clé | Type | Défaut | Description |
|---|---|---|---|
| `providers.<name>.enabled` | boolean | — | Activer/désactiver le provider |
| `providers.<name>.command` | string | — | Commande CLI à spawner |
| `providers.<name>.model` | string | — | Modèle à utiliser |
| `providers.<name>.maxTokens` | number | — | Limite de tokens par requête |
| `providers.<name>.costPer1k` | number | — | Coût par 1000 tokens |
| `providers.<name>.baseUrl` | string | — | Endpoint API OpenAI-compatible (active l'adapter HTTP) |
| `providers.<name>.apiKey` | string | — | Clé API pour l'authentification Bearer |

Quand `baseUrl` est défini, AIDEN utilise l'adapter API (`fetch`) au lieu du subprocess CLI. Le `command` est ignoré.

### Options pipeline

| Clé | Type | Défaut | Description |
|---|---|---|---|
| `pipeline.maxOutputCharsPerAgent` | number | 8000 | Nombre max de caractères par output d'agent injecté dans le prompt du suivant. |

### Options routing

| Clé | Type | Défaut | Description |
|---|---|---|---|
| `routing.strategy` | string | `prefer` | Stratégie : `fixed`, `prefer`, `auto`, `cost-optimized`. En mode `auto`, chaque agent reçoit le provider optimal. |
| `routing.fallback` | string[] | `[claude-code, gemini, opencode, copilot]` | Chaîne de fallback si le provider préféré est indisponible. |

### Options sécurité

| Clé | Type | Défaut | Description |
|---|---|---|---|
| `sanitize` | boolean | `true` | Active le filtrage des données sensibles (tokens, clés API, mots de passe, clés privées, connection strings) avant envoi aux providers. |

### Schéma projet (context.yaml)

Le scan de `aiden init` génère `.aiden/memory/context.yaml` avec ce schéma :

| Clé | Type | Description |
|---|---|---|
| `project.name` | string | Nom du projet (depuis le dossier) |
| `project.techStack` | string[] | Technologies détectées (node, typescript, react, nestjs, prisma, python...) |
| `project.architecture` | string[] | Infrastructure (docker, docker-compose, kubernetes) |
| `project.ci` | string[] | Systèmes CI/CD (github-actions, gitlab-ci, jenkins, circleci) |
| `project.packageManager` | string? | Gestionnaire de paquets (pnpm, yarn, npm, bun) |
| `project.databases` | string[]? | Bases de données (postgres, mysql, redis, mongodb) |
| `project.monorepo` | string? | Outil monorepo (turborepo, nx, lerna, pnpm-workspaces) |

Ces données sont injectées dans le prompt de chaque agent via la section `--- Project Context ---`.

## Schéma `skill.yaml`

Chaque skill dans `.aiden/skills/<name>/` contient un fichier `skill.yaml` :

```yaml
name: mon-skill                # (requis) Identifiant unique
version: 1.0.0                 # Version semver (defaut: 1.0.0)
description: ""                # Description lisible
author: equipe-platform        # Auteur (optionnel)
agents:                        # Agents cibles (defaut: tous)
  - dev
  - qa
modes:                         # Modes actifs (defaut: tous)
  - standard
  - enterprise
hooks:                         # Hooks declares
  on_feature_created:
    - command: ./hooks/notify.sh
      onFailure: warn
config:                        # Configuration custom
  jira_url: https://company.atlassian.net
priority: 10                   # Priorite (defaut: 0, plus haut = premier)
enabled: true                  # Actif (defaut: true)
```

| Clé | Type | Défaut | Description |
|---|---|---|---|
| `name` | string | (requis) | Identifiant unique, validé par Zod |
| `version` | string | `1.0.0` | Version semver |
| `description` | string | `""` | Description |
| `author` | string | — | Auteur |
| `agents` | AgentRole[] | tous | `lead`, `analyst`, `planner`, `dev`, `qa`, `docops` |
| `modes` | CTAMode[] | tous | `flash`, `standard`, `enterprise` |
| `hooks` | Record | — | Hooks par événement (même format que `aiden.config.yaml`) |
| `config` | Record | — | Paires clé-valeur libres |
| `priority` | integer | `0` | Tri décroissant |
| `enabled` | boolean | `true` | Désactiver sans supprimer |

Voir le [guide skills](/guides/skills) pour l'utilisation complète.

## Niveaux de surcharge

| Niveau | Fichier | Portée |
|---|---|---|
| 1. Défauts | (built-in) | Global |
| 2. Enterprise | `~/.aiden/enterprise.config.yaml` | Tous les projets |
| 3. Projet | `.aiden/aiden.config.yaml` | Ce projet |
| 4. Feature | `.aiden/features/<slug>/.override.yaml` | Cette feature |
| 5. CLI | `--mode`, `--provider` | Cette commande |

Chaque niveau écrase les valeurs du précédent. Les objets sont fusionnés en profondeur, les tableaux sont remplacés.
