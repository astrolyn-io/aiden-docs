# Glossaire

Agent, Retex, Quality gate... AIDEN a son vocabulaire. Voici le dictionnaire — chaque terme explique en une phrase.

---

## A

**ADR** — Architecture Decision Record. Document qui trace une decision architecturale significative, genere en mode enterprise.

**Agent** — Composant IA specialise dans un role (Lead, Analyst, Planner, Dev, QA, DocOps). Chaque agent herite de `BaseAgent` et recoit un provider par le routeur.

**Artefact** — Fichier Markdown avec frontmatter YAML genere par un agent (brief, spec, story, review, etc.). Stocke dans `.aiden/features/<slug>/`.

## B

**Brief** — Document de decouverte cree par l'Analyst. Resume la demande, le contexte, les risques et les contraintes.

**Bus de contexte** — Memoire partagee entre tous les agents (`.aiden/memory/context.yaml`). Contient la stack technique, l'architecture et le CI detectes.

## C

**Circuit breaker** — Mecanisme qui arrete un agent apres N echecs consecutifs et escalade a l'humain. Defaut : 3 tentatives.

**AIDEN** — AI-Driven Engineering. Le framework lui-meme.

## D

**Decisions log** — Fichier JSONL append-only qui trace toutes les decisions des agents. Protege par file locking.

## E

**Enterprise** — Mode le plus complet avec documentation, conformite et approbation humaine entre chaque phase.

## F

**Feature** — Unite de travail dans AIDEN. Chaque feature a son propre cycle de vie, son dossier d'artefacts et sa machine a etats.

**Flash** — Mode le plus rapide. Lead + Dev uniquement, pas de spec ni review. Ideal pour les hotfixes.

## L

**Lead** — Agent orchestrateur qui coordonne tous les autres agents. Detecte aussi le mode optimal via l'analyse de signaux.

## M

**Mode** — Niveau de complexite du workflow : flash, standard ou enterprise. Determine quels agents sont executes et dans quel ordre.

## O

**OpenCode** — Provider CLI open source alternatif. Outil agentic configurable pointant vers n'importe quel endpoint.

## P

**Provider** — Service IA externe (Claude Code, Gemini, OpenCode, Copilot ou API custom) utilise par les agents. Deux types d'adapters : CLI (subprocess) et API (fetch).

## Q

**Quality gate** — Verification automatique entre deux phases du workflow. Bloque la progression si les criteres ne sont pas remplis.

## R

**Retex** — Retour d'experience structure (lessons learned). Stocke dans `.aiden/retex/` avec contexte, lecon et recommandation. Injecte dans les prompts agents sous `--- Lessons Learned ---`.

**Routage** — Selection automatique du provider optimal selon la tache. Quatre strategies : `fixed`, `prefer`, `auto`, `cost-optimized`.

## S

**Sanitizer** — Mecanisme de filtrage qui masque les donnees sensibles (tokens, cles API, mots de passe, connection strings) dans les prompts avant envoi aux providers.

**Skill** — Package declaratif (YAML + Markdown) dans `.aiden/skills/` qui injecte instructions, regles et hooks dans le pipeline AIDEN. Mecanisme d'extensibilite principal pour les entreprises.

**Spec** — Specification technique allegee creee par le Planner. Contient les exigences et les criteres d'acceptation.

**Standard** — Mode intermediaire avec analyse, planification, developpement et review. Le choix par defaut.

**Story** — User story decomposee par le Planner avec criteres d'acceptation. Numerotees sequentiellement (001-*, 002-*).

## T

**Token budget** — Limite de tokens par feature et par mois pour controler les couts. Configurable par mode (flash: 5000, standard: 20000, enterprise: 100000).
