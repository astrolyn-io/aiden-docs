# Glossaire

> **Ce que vous allez apprendre**
> - La terminologie AIDEN
> - Les acronymes et concepts clés

## A

**ADR** — Architecture Decision Record. Document qui trace une décision architecturale significative.

**Agent** — Composant IA spécialisé dans un rôle (Lead, Analyst, Planner, Dev, QA, DocOps).

**Artefact** — Fichier Markdown avec frontmatter YAML généré par un agent (brief, spec, story, review, etc.).

## B

**Brief** — Document de découverte créé par l'Analyst. Résume la demande et le contexte.

**Bus de contexte** — Mémoire partagée entre tous les agents (`.aiden/memory/context.yaml`).

## C

**Circuit breaker** — Mécanisme qui arrête un agent après N échecs consécutifs et escalade à l'humain.

**AIDEN** — AI-Driven Engineering. Le framework lui-même.

## D

**Decisions log** — Fichier JSONL append-only qui trace toutes les décisions des agents.

## E

**Enterprise** — Mode le plus complet avec documentation, conformité et approbation humaine.

## F

**Feature** — Unité de travail dans AIDEN. Chaque feature a son propre cycle de vie et dossier d'artefacts.

**Flash** — Mode le plus rapide. Lead + Dev uniquement, pas de spec ni review.

## L

**Lead** — Agent orchestrateur qui coordonne tous les autres agents.

## M

**Mode** — Niveau de complexité du workflow : flash, standard ou enterprise.

## O

**OpenCode** — Provider CLI open source alternatif. Outil agentic configurable pointant vers n'importe quel endpoint.

## P

**Provider** — Service IA externe (Claude Code, Gemini, OpenCode, Copilot ou API custom) utilisé par les agents.

## Q

**Quality gate** — Vérification automatique entre deux phases du workflow.

## R

**Retex** — Retour d'expérience structuré (lessons learned). Stocké dans `.aiden/retex/` avec contexte, leçon et recommandation. Injecté dans les prompts agents.

**Routage** — Sélection automatique du provider optimal selon la tâche.

## S

**Sanitizer** — Mécanisme de filtrage qui masque les données sensibles (tokens, clés API, mots de passe, connection strings) dans les prompts avant envoi aux providers.

**Skill** — Package déclaratif (YAML + Markdown) dans `.aiden/skills/` qui injecte instructions, règles et hooks dans le pipeline AIDEN. Mécanisme d'extensibilité principal pour les entreprises.

**Spec** — Spécification technique allégée créée par le Planner.

**Standard** — Mode intermédiaire avec analyse, planification et review.

**Story** — User story décomposée par le Planner avec critères d'acceptation.

## T

**Token budget** — Limite de tokens par feature et par mois pour contrôler les coûts.
