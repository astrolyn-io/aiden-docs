# Référence Artefacts

> **Ce que vous allez apprendre**
> - Les 7 types d'artefacts AIDEN
> - Le format frontmatter YAML
> - Un exemple réel de chaque artefact

## Format commun

Chaque artefact est un fichier Markdown avec frontmatter YAML :

```yaml
---
type: spec
feature: export-csv
version: 1
created: 2026-03-26
author: planner
status: draft
provider: claude-code
tokensUsed: 3421
mode: standard
---
```

Statuts possibles : `draft`, `review`, `approved`, `implemented`.

## Les 7 artefacts

### brief.md

Généré par l'**Analyst**. Résume la demande, contexte, objectif, contraintes et risques.

### spec.md

Généré par le **Planner**. Spécification allégée (1-2 pages) avec exigences et critères d'acceptation.

### story.md (dans stories/)

Généré par le **Planner**. User stories numérotées (001-*, 002-*) avec critères d'acceptation.

### review.md

Généré par le **QA**. Résultat du code review avec score, issues et suggestions.

### change-summary.md

Généré par le **Dev** (mode flash). Résumé des changements effectués.

### architecture-decision.md

Généré par le **Planner** (mode enterprise). ADR : contexte, décision, alternatives, conséquences.

### quality-report.md

Généré par le **QA** (mode enterprise). Rapport complet avec métriques, sécurité et recommandations.

## Emplacement

Tous les artefacts sont dans `.aiden/features/<feature-slug>/`.
